# Docs

Fission is easier to learn when you stay in one lane at a time.

## Learn

Use the Learn lane when you need the framework model first:

- what the shared app/runtime contract is,
- how widgets become layout and paint output,
- which examples and target paths are already proven in this repo.

Start with [Learn overview](/docs/learn/overview).

## Guides

Use Guides when you are implementing a subsystem in a real app:

- app structure,
- async resources and capabilities,
- input, text, IME, and environment sync,
- theming, i18n, media, portals, 3D, shells, and diagnostics.

Start with [App structure](/docs/guides/app-structure).

## Cookbook

Use Cookbook when you want a practical recipe backed by checked-in code:

- build a counter,
- wire a capability,
- keep a timer or service alive,
- add platform targets,
- drive a live UI test.

Start with [Build a counter](/docs/cookbook/build-a-counter).

## Reference

Use Reference when you need the exact public surface:

- core runtime types,
- shell and CLI contracts,
- subsystem summaries,
- widget pages grouped by job.

Open [Reference overview](/reference/overview/overview).

## Repo-backed pages outside the docs lanes

- [Examples](/examples) maps the checked-in example crates.
- [Playground](/playground) shows the fastest local loops.
- [Showcase](/showcase) highlights the most product-like samples in the repo.

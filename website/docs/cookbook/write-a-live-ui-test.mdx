# Write a live UI test

The repo uses `fission-test-driver::LiveTestClient` across many examples.

## Typical flow

1. launch the app with `FISSION_TEST_CONTROL_PORT=<port>`,
2. wait for readiness,
3. interact through text, coordinates, or key presses,
4. capture screenshots or semantic tree data,
5. shut the app down cleanly.

## Minimal sketch

```rust
let client = LiveTestClient::connect(control_port);
client.wait_for_ready(15_000)?;
client.tap_text("Increment")?;
client.assert_text_visible("Count: 1")?;
client.screenshot("/tmp/counter.png")?;
client.quit()?;
```

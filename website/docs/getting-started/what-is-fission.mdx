# What is Fission

Fission is a Rust-first framework for building product interfaces that run on desktop and web, including responsive mobile web workflows, from the same source model. It is useful when teams need predictable behavior and a single architecture rather than multiple platform-specific UI stacks.

On this website we position Fission for desktop and web platforms today. Android and iOS are not yet natively supported.

A Fission app has a simple flow.

- You define a typed application state.
- You build a widget tree from that state.
- The runtime evaluates actions and reducers to update state.
- The renderer converts that tree into layout, paint, and pixels.

That flow is repeated many times a frame, which is why consistency matters. Teams usually feel the value after a few iterations because every behavior change is represented in one place.

If you are used to adding platform-specific code for layout and event handling, Fission reduces that split: state, behavior, and rendering contracts stay together.

When reading the docs, stay in this order:

1. install the toolchain,
2. build one tiny app,
3. learn how widgets compose layout,
4. understand action and reducer flow,
5. add async behavior with commands, services, and jobs.

This keeps each concept connected to the next.

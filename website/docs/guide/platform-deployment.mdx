# Platform and deployment

Use one architecture and one behavior model, then apply platform output steps.

Recommended path:

- implement and stabilize app behavior,
- validate desktop,
- validate web/WASM,
- validate iOS simulator,
- validate Android emulator,
- confirm behavior and data consistency across environments.

Keep environment-specific values out of reducer logic. Put them in deployment configuration and target descriptors.

This pattern is what helps behavior remain consistent as your target list grows.

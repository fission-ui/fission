# Rendering pipeline

Each frame follows:

1. `Widget::build()` produces `Node`
2. `CoreIR` lowering
3. layout (`LayoutSnapshot`)
4. paint list / render scene
5. backend swapchain output

## Why it matters

The best place to debug a regression is the stage where output diverged:

- wrong text only after action dispatch -> reducer or build,
- wrong size/position -> layout,
- wrong pixels with stable layout -> paint/style,
- target-specific stutter -> frame/backend tuning.

## Observability

`FISSION_DIAG` supports per-stage captures and is your fastest path for stage-level triage.

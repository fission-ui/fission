# Widget trait

Every renderable unit implements the same minimal contract:

```rust
fn build(&self, ctx: &mut BuildCtx<S>, view: &View<S>) -> Node
```

This keeps rendering pure and predictable. `build()` is not a place for side effects; it is a projection from state to nodes.

## What this means in practice

- all behavior starts from state,
- action dispatch stays explicit,
- async and I/O stay outside build through contexts and effects.

## How to reason about a widget implementation

1. Read only what is in `view.state`.
2. Bind any actions your UI needs.
3. Return a deterministic `Node` tree.

If side effects leak into build, tests become noisy and replay behavior degrades.

## Common usage pattern

```rust
impl Widget<MyAppState> for Home {
    fn build(&self, ctx: &mut BuildCtx<MyAppState>, view: &View<MyAppState>) -> Node {
        // map state -> layout -> interaction
        Column::default().into_node()
    }
}
```

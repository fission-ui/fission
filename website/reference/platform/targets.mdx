# Targets

Targeting is not a separate architecture. It is packaging and runtime-specific output around the same app model.

Current output targets are desktop shells, web/WASM, iOS, and Android.

- **desktop**: standard Vello-based runtime for Windows, macOS, and Linux
- **web**: WASM host from `platforms/web`
- **ios**: iOS simulator workflow from `platforms/ios`
- **android**: Android emulator workflow from `platforms/android`

Use the same state/actions model and keep target-specific differences at the runtime/config boundary.

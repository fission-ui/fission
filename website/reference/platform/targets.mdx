# Targets

Targets are generated host layers around the same app model.

Supported targets for scaffolds are:

- `windows`, `macos`, `linux`
- `web`
- `ios`
- `android`

## Shared behavior, target-specific launch

Keep `src/app.rs` as your product source. Generated files under `platforms/<target>/` control
packaging and launch commands.

## Typical commands

- desktop: `cargo run`
- web: `cargo fission add-target web && ./platforms/web/run-browser.sh`
- iOS: `cargo fission add-target ios && ./platforms/ios/run-sim.sh`
- Android: `cargo fission add-target android && ./platforms/android/run-emulator.sh`

## What to verify per target

- action/reducer consistency,
- semantics discovery,
- layout behavior under different viewport sizes,
- baseline performance.

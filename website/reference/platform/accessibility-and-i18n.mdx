# Accessibility and i18n

Treat these as product requirements from the first feature.

## Accessibility

Attach semantics to interactive nodes where users need clear intent signals:

- `role`
- `label`
- `focusable`

Use role and label checks in tests rather than visual-only assertions.

## Internationalization

Use stable keys (`TextContent::Key`) with per-locale bundles, and switch locale through environment sync
(`with_sync_env`).

## Practical checklist

- every primary interactive control has role and label,
- dynamic strings come from keys,
- long strings are checked for clipping/overflow,
- locale switch preserves layout flow.

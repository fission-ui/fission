# Testing

Use this order to keep fixes localized:

1. **Reducer tests**: verify state transitions.
2. **Effect input tests**: verify completion payload decoding.
3. **Headless widget tests**: verify behavior without platform dependencies.
4. **Live target smoke**: verify actual runtime integration.

## Tools

- `fission_test::TestHarness` and `TestDriver` for deterministic checks,
- `fission_test_driver::LiveTestClient` for target-level trees and interaction checks,
- diagnostics env vars (`FISSION_DIAG_*`) for stage-based traces.

## Good failure signals

- reducer outputs match expected state,
- action inputs decode cleanly,
- role/text visibility remains consistent,
- async lifecycle completes with expected state transitions.

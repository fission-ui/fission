# Button

Use it when the user needs to interact, choose a value, or submit intent.


This is part of the Fission widget API and belongs in the same typed, state-driven model used across the rest of the framework.


**When you need it**
- This widget is best when the control drives explicit `Action` dispatch or receives values from a reducer-owned state model.
- Use predictable state updates as your entry point for all behavior changes.
- Combine with semantic labels and role attributes for reliable interaction testing.

## Minimal usage
```rust
use fission_core::ui::Button;

let node = Button {
    ..Default::default()
}.into_node();
```

## API surface
- `id`: `Option<NodeId>`
- `child`: `Option<Box<Node>>`
- `on_press`: `Option<ActionEnvelope>`
- `semantics`: `Option<Semantics>`
- `width`: `Option<f32>`
- `height`: `Option<f32>`
- `min_width`: `Option<f32>`
- `max_width`: `Option<f32>`
- `flex_grow`: `f32`
- `flex_shrink`: `f32`
- `padding`: `Option<[f32; 4]>`
- `style`: `Option<ButtonStyleOverride>`

## Implementation checks
- Keep control values in state, validate there, and surface results through completion actions or reducer transitions.
- Keep callbacks/closures small and deterministic when possible, then move side effects to async boundaries.
- If the widget affects layout or input, verify with both state transitions and target-level smoke tests.

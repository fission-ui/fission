# Container

Use it to control how child nodes are arranged, measured, and constrained.


This is part of the Fission widget API and belongs in the same typed, state-driven model used across the rest of the framework.


**When you need it**
- This widget is best when you need stable structure before styling. Keep layout decisions explicit so state and constraints stay predictable.
- Use predictable state updates as your entry point for all behavior changes.
- Combine with semantic labels and role attributes for reliable interaction testing.

## Minimal usage
```rust
use fission_core::ui::Container;

let node = Container {
    ..Default::default()
}.into_node();
```

## API surface
- `id`: `Option<NodeId>`
- `child`: `Option<Box<Node>>`
- `width`: `Option<f32>`
- `height`: `Option<f32>`
- `min_width`: `Option<f32>`
- `max_width`: `Option<f32>`
- `min_height`: `Option<f32>`
- `max_height`: `Option<f32>`
- `padding`: `[f32; 4]`
- `flex_grow`: `f32`
- `flex_shrink`: `f32`
- `background_fill`: `Option<Fill>`

## Implementation checks
- Build with the state model first, then layer spacing, alignment, and visual treatment. Layout mistakes are usually cheaper to fix before async behavior is introduced.
- Keep callbacks/closures small and deterministic when possible, then move side effects to async boundaries.
- If the widget affects layout or input, verify with both state transitions and target-level smoke tests.

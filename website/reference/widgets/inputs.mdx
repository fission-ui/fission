# Input widgets

Input widgets should map to explicit actions first and async work second.

## Common controls

- `Button`
- `Checkbox`
- `Radio`
- `Slider`
- `Switch`
- `TextInput`
- `NumberInput`
- `Select` / `SelectItem`
- `Combobox`
- `SegmentedControl`
- `DatePicker`, `DateRangePicker`, `TimePicker`

## Drag, drop, and lower-level input

- `GestureDetector`
- `Draggable`
- `DragTarget`
- `Dropzone`
- `Editable`

## Form composition helpers

- `FormControl`
- `DropDown`
- `FileUpload`
- `RangeSlider`

Pair this section with the text-lab example when the hard part is input behavior rather than layout.

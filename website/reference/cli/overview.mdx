# CLI overview

The CLI flow is intentionally small:

1. `fission init <app>` creates a runnable scaffold.
2. `cargo fission add-target <target>` adds host output for a runtime.

Example:

```bash
cargo install fission-cli
fission init my-app
cd my-app
cargo run
cargo fission add-target web ios android
```

Generated targets are launched from each `platforms/<target>/` directory. Keep generated artifacts committed only where that is part of your team practice; otherwise use local scripts for iteration and source control for product decisions.

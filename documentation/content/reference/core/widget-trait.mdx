# Widget authoring boundary

The authoring boundary is `From<Component> for Widget`. A component is an ordinary Rust struct. To make it renderable, convert that struct into the closed `Widget` tree.

```rust
use fission::prelude::*;

pub struct SaveDraftButton;

impl From<SaveDraftButton> for Widget {
    fn from(component: SaveDraftButton) -> Self {
        let (ctx, _view) = fission::build::current::<ComposerState>();
        let save = with_reducer!(ctx, SaveDraft, save_draft);

        Button {
            on_press: Some(save),
            child: Some(Text::new("Save draft").into()),
            ..Default::default()
        }
        .into()
    }
}
```

Use `.into()` at composition boundaries. Child slots such as rows, containers, overlays, and custom widgets store `Widget`, including `Widget`, `Vec<Widget>`, or `Option<Widget>` depending on the widget.

`BuildCtxHandle` and `ViewHandle` are still available through `fission::build::current::<State>()` when a component needs state, environment, or reducer wiring. Keep conversion deterministic: read state, derive the widget tree, and avoid unrelated side effects.
